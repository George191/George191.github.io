问题汇总

“uwsgi” 日志中如果出现：* no app loaded. going in full dynamic mode *

启动app失败 （网上说是安装 “uwsgi” 的方式不对 必须用 “pip”）

“uwsgi” 日志中如果出现 “python” 版本和自己本地的版本不一致

加载错了环境，导致找不到依赖或者py包的异常

执行 “uwsgi” 出现 chdir(): No such file or directory [core/uwsgi.c ling…

查看 “xml, ini” 配置问题,或者结束掉 “uwsgi” 进程重新开启

执行 “uwsgi” 控制台抛异常 缺失文件

which uwsgi 找到 “uwsgi” 目录 “ldd” 这个目录 看是否有 “not found” 有的话 “which” 出来加环境

nginx：[error] invalid PID number/ 或者找不到PID文件

启动 “nginx” 同时指定 “nginx.conf” 路径 ./nginx -c xxx/nginx.conf

注： 结束进程命令

pkill -9 nginx
pkill -9 uwsgi
